# Change Proposal: align-session-history-list-layout

Change-ID: align-session-history-list-layout

概要

セッション履歴画面（History）およびステータス画面（Status）でリスト項目やバッジ（例: "- First Victory"）の横位置が他のラベル付き行と揃っておらず、UIの見た目が崩れている。リストのハイフン（`-`）や箇条書き項目が、CSSのflexのように隣接テキストと同じ行に並んでしまうケースがあるため、すべての画面でラベルと値、箇条のインデントを統一して揃える仕様を追加する。

目的

- ユーザー情報やセッション履歴の可読性向上
- UI コンポーネントの表記整合性確保
- 将来的な画面追加時に同じ整列ルールを再利用可能にする

スコープ

- 影響範囲: `internal/ui/history.go`, `internal/ui/status.go` の表示ロジックと共通スタイル（`internal/ui/components` 配下にヘルパーを追加する案）
- 変更種別: 表示（UI）の仕様追加 / 小さなリファクタ（共通描画ヘルパー）
- 実装は本提案の承認後に行う（本段階では実装不要）

非目的

- 大きなレイアウトフレームワークの導入や外部ライブラリの採用
- 既存の履歴データモデルの変更

利点

- 画面ごとに手作業で揃える必要がなくなる
- アクセシビリティと安定したテキストレンダリングが得られる

リスク & トレードオフ

- 既存コードに手を入れるため、小さなマージ作業が発生する
- コンソール幅やヘッダー幅に依存するため、端末サイズによっては折り返しが発生する（現状の lipgloss を使ったレイアウトと同様）

1. 仕様（spec delta）作成 — この提案に含む（完了）
2. 実装タスクを `tasks.md` に落とす（本提案に含む）
3. 実装 & テスト & レビュー（実装フェーズ）
